# 2026-06-05 진입 신호 스윕 — 풀백 전략군 수익 구간 없음 (V1 눌림폭 하한만 리드)

## 배경

[청산 스윕](2026-06-05-exit-sweep-signal-problem-confirmed.md)에서 어떤 TP/SL도 신호를 구제하지
못함을 확인했다. 남은 레버는 진입 신호다. parity 하니스에 **진입 변형 스윕**(`EntrySweepTest`)을 추가해
가설 기반 진입 필터 12종을 같은 조건(시장가 진입, 고정 청산 TP+3/SL−2)에서 비교했다.

- 실행: `./gradlew test -Dbacktest.entrysweep=true --tests "*EntrySweepTest"`
- 엔진: 운영 Java 리플레이, 시장가 진입(maker 역선택 배제)
- 비용: 각 leg 0.05%. 분리: train 120d / test 60d
- 판정 기준: **train** PFgross가 ≥~1.15로 올라가되 거래수 붕괴 없을 것(일반화되는 엣지)

## 해석

- **일반화되는 gross 엣지 ≥1.0인 변형 없음.** baseline train 0.909에서 의미 있게 오른 건 V1(0.974)뿐.
- **V1(최소 눌림폭 0.5% 요구)** = 유일한 리드. 고점 바로 아래 추격 진입을 걸러 train≈test(0.97/0.96)로
  균형까지 좋다. 다만 여전히 breakeven 미달, net ~0.85(비용 후 적자).
- **V11**: train 1.073으로 1.0 돌파하나 test 0.811 — train≫test = OOS 실패(574 trade에 3필터 스택한 과적합).
- V2(minDist 1.0)·V4(consec3)의 test 우위는 신호 붕괴(328·1089) 또는 test 윈도우 운으로, 신뢰 불가.

## 결론 / 다음

- **풀백 반등 전략군은 이 180일 Upbit 데이터에서 수익 구간이 없다.** 진입 파라미터 미세조정으로는
  breakeven을 못 넘는다. 실거래 전환 금지선(ADR-001) 유효.
- 남은 방향 두 갈래:
  1. **V1 눌림폭 하한**을 출발점으로, 운영과 다른 *새 피처*(다중 타임프레임 확인, BTC/시장 레짐 게이트,
     호가/유동성, 변동성 정규화 사이징)를 더해 train gross ≥1.15 영역을 탐색.
  2. 풀백군 폐기 후 다른 전략 가설로 전환.
- 어느 쪽이든 이 하니스의 PFgross(1차 게이트) → PFnet train/test OOS(채택 게이트)로 판정.
