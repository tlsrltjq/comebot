# Project Next Steps

## 1단계: 자동 실행 운영 UX 정리

목표:

- Telegram 후보 조회와 실행 결과를 더 짧게 요약한다.
- scheduler 상태와 알림 상태를 한 화면에서 확인하기 쉽게 정리한다.
- 설정 변경 기능은 만들지 않는다.

## 2단계: 중복 진입 제한 강화

목표:

- 보유 포지션뿐 아니라 최근 실행 이력을 기준으로 재진입 쿨다운을 둔다.
- market별 쿨다운 설정을 지원한다.

## 3단계: 운영 기록 점검

목표:

- scheduler 요약과 history 기록이 서로 맞는지 점검한다.
- 알림 실패가 실행 결과를 바꾸지 않는지 회귀 테스트를 유지한다.

## 보류: 실제 주문 API
