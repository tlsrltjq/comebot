# 2026-06-05 다중 TF 확인 — 코인 1h 정렬은 OOS 최선이나 천장 0.97, 풀백군 엣지 부재 확정

## 배경

레짐 게이트가 F1(BTC≠DOWN)에서 소진된 뒤, 게이트가 아닌 **신호 보강** — 매매 코인 자신의 상위 TF(1h)
추세가 1m 풀백과 정렬되는지(falling knife 회피) — 를 검증했다.

- 도구: `MultiTimeframeTest`(`-Dbacktest.mtf=true`), `RegimeContext.coinHourlyTrend/coinReturnPct`(1m→1h 리샘플)
- 고정: 시장가 진입, TP+3/SL−2, V1 진입, train 120d/test 60d

## STEP 1 — 코인 1h 추세별 분해 (gross PF)

| 버킷 | TRAIN n / PFg | TEST n / PFg |
|---|---|---|
| coin1h DOWN | 667 / 0.882 | 202 / 0.944 |
| coin1h UP | 613 / 0.992 | 423 / 0.950 |

→ BTC와 같은 패턴(상위 TF UP가 DOWN보다 나음). 가설 방향 확인. 단 UP train 0.992로 1.0 미달.

## 판정

- **M1(coin 1h==UP)이 단일 필터 중 OOS 최선**: test 0.976 ≥ train 0.967(균형·과적합 없음), 거래 1240(F1보다 많음),
  코인 쏠림 없음, MDD 반감. F1과 동급 이상.
- **그러나 게이트 ≥1.05 미달**: 최선도 gross 0.97, net 0.857(적자).
- **결합(F1&M1, F1&M2)은 또 OOS 악화**(test 0.918/0.909, train만 0.993) — breadth 결합과 동일한 감산. 방향성 필터 스택은 train 과적합/test 붕괴.

## 메타 결론 (중요)

지금까지 시도한 모든 레버가 같은 결과로 수렴:

| 레버 | 최선 gross(all) |
|---|---|
| 청산 TP×SL 12칸 | train 전부 <0.94 |
| 진입 변형 12종 | 0.974(V1) |
| 레짐 게이트 6+ | 0.969(F1) |
| 다중 TF 확인 6 | 0.970(M1) |

→ **V1 풀백 반등군은 이 180일 Upbit 데이터에서 tradeable 엣지가 없다.** 어떤 필터·확인·청산도 gross를
0.94~0.97 밴드 위(≥1.0)로 못 올린다. 신호의 미세 음의 엣지를 필터가 다듬을 뿐 부호를 못 바꾼다.
또한 **방향성 필터 2개 이상 스택은 일관되게 OOS를 깬다**(F1&M1, F1&breadth) — 확정된 패턴.
